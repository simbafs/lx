## What is LX?

LX is a **declarative layout syntax** that describes geometric relationships between HTML elements using HTML attributes. Instead of specifying concrete pixel values, you describe _how elements relate to each other_.

## Core Principles

### 1. Geometry First

### 2. Auto-inference

### 3. Zero Runtime Dependencies

Pure Vanilla JavaScript. No build tools, no framework dependencies.

## Quick Comparison

| Feature      | CSS                           | LX                         |
| ------------ | ----------------------------- | -------------------------- |
| Positioning  | Manual coordinate calculation | Declarative relationships  |
| Responsive   | Media queries                 | Variables + expressions    |
| Dependencies | Manual ordering               | Automatic topological sort |
| Updates      | Manual DOM manipulation       | Call `lx.update()`         |

## When to Use LX

LX is ideal for:

- **Dashboard layouts** with fixed header, sidebar, and content areas
- **Card grids** with consistent spacing and aspect ratios
- **Form layouts** with precise label-input alignment
- **Any layout where elements depend on each other's positions**

LX may not be the best choice for:

- Highly fluid layouts that need to reflow naturally
- Complex grids with variable column spans
- Animations or transitions between states
